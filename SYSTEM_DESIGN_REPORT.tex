\section{What is UniLodge v2?}

\begin{tcolorbox}[highlight, title=Platform Summary]
UniLodge v2 is an enterprise-grade accommodation management platform designed specifically for academic institutions. It connects students and visiting scholars with verified room listings across universities, offering a complete digital lifecycle from discovery to booking and checkout.
\end{tcolorbox}

The platform is built on modern technologies:
\begin{itemize}
  \item \textcolor{Blue}{\textbf{Frontend:}} Next.js 14 with React 19 and Tailwind CSS
  \item \textcolor{Green}{\textbf{Backend:}} Express.js REST API with Node.js
  \item \textcolor{Purple}{\textbf{Database:}} MongoDB Atlas cloud hosting
  \item \textcolor{Orange}{\textbf{AI:}} OpenRouter API integration
\end{itemize}

\section{Core Features}

\subsection{Room Discovery}
\begin{itemize}
  \item 13 pre-seeded rooms across 6 universities
  \item Advanced filtering by university, type, price, and amenities
  \item Real-time search with 500ms debounce
  \item Full-text search on descriptions
\end{itemize}

\subsection{Booking Management}
\begin{itemize}
  \item Complete booking lifecycle (Pending → Confirmed → Checked In → Checked Out)
  \item Payment tracking (paid/unpaid/refunded)
  \item Check-in and checkout tracking with timestamps
  \item Full booking history per user
\end{itemize}

\subsection{User Roles}
\begin{itemize}
  \item \textbf{Guest:} Browse and book rooms
  \item \textbf{Warden:} Manage assigned building and rooms
  \item \textbf{Admin:} Full system administration and analytics
\end{itemize}

\subsection{AI Features}
\begin{itemize}
  \item OpenRouter LLM integration for chat assistance
  \item Dynamic price suggestions based on market analysis
  \item Semantic search with embeddings
  \item Personalized recommendations
\end{itemize}

\section{Three-Tier Architecture}

The system is organized into three logical tiers:

\subsection{Tier 1 --- Presentation Layer}
\textcolor{Blue}{\textbf{Next.js 14 + React 19 + Tailwind CSS}}
\begin{itemize}
  \item Port: 3002
  \item Glassmorphism UI design
  \item Responsive for mobile, tablet, and desktop
  \item Real-time data synchronization
  \item Client-side state management with React hooks
\end{itemize}

\subsection{Tier 2 --- Application Layer}
\textcolor{Green}{\textbf{Express.js + Node.js}}
\begin{itemize}
  \item Port: 3001
  \item 15+ REST API endpoints
  \item JWT token authentication (7-day expiry)
  \item Rate limiting (100 requests per minute)
  \item Input validation with Zod
  \item CORS enabled for frontend communication
\end{itemize}

\subsection{Tier 3 --- Data Layer}
\textcolor{Purple}{\textbf{MongoDB Atlas}}
\begin{itemize}
  \item Cloud-hosted MongoDB database
  \item Three main collections: Users, Rooms, Bookings
  \item Composite indexes for efficient queries
  \item Connection pooling for performance
  \item 99.9\% uptime SLA
\end{itemize}

\section{Technology Stack}

\subsection{Frontend Stack}
\begin{itemize}
  \item Next.js 14 --- React framework with SSR
  \item React 19 --- UI component library
  \item TypeScript 5.3 --- Type safety
  \item Tailwind CSS --- Utility-first styling
  \item Lucide React --- Icon library
\end{itemize}

\subsection{Backend Stack}
\begin{itemize}
  \item Node.js 20+ --- JavaScript runtime
  \item Express.js 4.18 --- Web framework
  \item TypeScript --- Type safety
  \item Zod 3.22 --- Schema validation
  \item CORS --- Cross-origin support
\end{itemize}

\subsection{Database Stack}
\begin{itemize}
  \item MongoDB 6.0+ --- Document database
  \item Mongoose 9.5 --- ODM (Object Data Modeling)
  \item MongoDB Atlas --- Cloud hosting
\end{itemize}

\subsection{Security and AI Stack}
\begin{itemize}
  \item JWT --- Token-based authentication
  \item bcryptjs --- Password hashing (10 salt rounds)
  \item OpenRouter API --- AI model integration
  \item Axios --- HTTP client
\end{itemize}

\section{Collections Overview}

The database consists of three main collections:

\subsection{Users Collection}

Stores user information and authentication data.

\begin{lstlisting}[language=json]
{
  _id: ObjectId,
  name: String,
  email: String (unique),
  password: String (hashed),
  role: Enum ['GUEST', 'WARDEN', 'ADMIN'],
  createdAt: Date
}
\end{lstlisting}

\subsection{Rooms Collection}

Stores room listings with details and availability.

\begin{lstlisting}[language=json]
{
  _id: ObjectId,
  roomNumber: String (unique),
  type: Enum ['Single', 'Double', 'Suite'],
  price: Number,
  amenities: [String],
  rating: Number,
  isAvailable: Boolean,
  university: String,
  createdAt: Date
}
\end{lstlisting}

\subsection{Bookings Collection}

Stores booking records with status information.

\begin{lstlisting}[language=json]
{
  _id: ObjectId,
  roomId: ObjectId,
  userId: ObjectId,
  checkInDate: Date,
  checkOutDate: Date,
  status: Enum ['Pending', 'Confirmed'],
  totalPrice: Number,
  createdAt: Date
}
\end{lstlisting}

\chapter{Security Implementation}

\section{Authentication}

\begin{itemize}
  \item JWT tokens with 7-day expiry
  \item bcryptjs password hashing (10 salt rounds)
  \item Tokens stored in localStorage
  \item Verified on protected routes
\end{itemize}

\section{Protection Mechanisms}

\begin{itemize}
  \item Rate limiting: 100 requests per minute per IP
  \item Input validation with Zod schemas
  \item CORS configuration for whitelisted origins
  \item Secure password comparison (constant-time)
\end{itemize}

\chapter{Installation and Setup}
% ═══════════════════════════════════════════════════════════

\section{Backend Setup}

\begin{lstlisting}[language=bash]
cd apps/backend
npm install

# Create .env file with configurations
npm run seed    # Seeds database with sample data
npm run dev     # Starts on http://localhost:3001
\end{lstlisting}

\section{Frontend Setup}

\begin{lstlisting}[language=bash]
cd apps/frontend
npm install

# Configure API URL
echo "NEXT_PUBLIC_API_URL=http://localhost:3001/api" > .env.local

# Start development server
npm run dev     # Starts on http://localhost:3002
\end{lstlisting}

\section{Accessing the Application}

\begin{itemize}
  \item Frontend: \texttt{http://localhost:3002}
  \item Backend API: \texttt{http://localhost:3001/api}
  \item Test accounts provided by seed script
\end{itemize}
